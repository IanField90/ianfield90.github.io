More recently applications have a few 'run-once' screens, which can help educate the user to some of the app features, or guide them through a more complicated process. 

One pro for using the emulator is that it is possible to retrieve a SQLite database that is used by the application.


Stetho essentially allows you to peak inside the current application's state. Be it what's stored in SharedPreferences, or a SQLite databse. But what's more it allows you to interact and alter the shared preferences, and query the database as it were live, but without the added hassle of attaching the debugger at a predetermined point, and manually writing surplus code, or executing and evaluating expressions. Sometimes you know the query that you want to write, but you don't want to have to relaunch the applications.

Stetho, by Facebook, utilises Chrome's Inspector tool.


Chrome inspector requires just 1 line altered in your application's code in order to debug a WebView on a live device.

Something to note: It appears to be possible to debug a WebView with or without this line on an emulator. It took me a bit of head scratching until I realised this!
